# 频道哨兵 Channel Sentinel 使用说明

本说明仅覆盖软件界面中的菜单与按钮操作，帮助你快速上手完成检测、管理与导出。

---

## 登录页
- 用户名/密码输入框：默认 admin/admin（建议登录后在“修改密码”中更新）
- 验证码图片：点击图片刷新验证码
- 登录按钮：提交用户名、密码与验证码后进入系统
- 退出登录：页面右下角或设置中可执行退出，返回登录页

---

## 首页（检测与导入）

### 顶部导航与版本
- 检测结果：跳转到结果页查看与管理检测记录
- 版本信息：右下角版本徽标，点击打开版本弹窗
  - 立即更新：本地源码模式会执行更新；Docker 环境给出拉取镜像指引

### 版本管理（当前页面）
- 保存（小硬盘图标）：保存当前数据到 /data/streams.json，并生成时间戳版本文件
- 加载（文件夹图标）：加载选中的时间戳版本为当前数据
- 刷新（旋转箭头）：刷新版本列表
- 删除（红色垃圾桶）：删除选中的时间戳版本文件

### UDPXY/rtp2httpd 服务器
- 添加服务器（加号）：向列表新增服务器地址
- 删除当前服务器（垃圾桶）：删除选中的服务器
- 下拉选择：选择当前使用的服务器（影响内网导出基址）

### 组播范围与CIDR
- 输入 CIDR 范围（例如 239.1.1.0/24）
- 应用CIDR（箭头圆圈）：将范围应用到批量生成通道

### 导入与上传
- 本地加载（加载文件）：选择本地 txt/m3u 文件并解析填充
- 本地上传（上传文件）：将文件内容导入为列表
- 网络加载：输入远程 txt/m3u 链接，抓取并解析

### 检测控制
- 开始检测（绿色“开始检测”）：对当前列表批量调用 ffprobe 检测
- 停止检测（红色“停止检测”）：终止正在进行的检测任务

### 导出与过滤
- 导出下拉：生成 M3U/TXT/JSON（支持内/外网、筛选在线/离线）
- 过滤按钮：全部 / 在线 / 离线
- 批量操作：删除（红色按钮） / 清空（灰色按钮）
- 分页：上一页 / 下一页

### 通道编辑弹窗
- 字段编辑：名称、tvg-id/tvg-name、Logo、分组、时移（catchup）
- 分组选择与新增：从下拉选择或输入“新分组”后点击“添加”
- 台标模板：输入模板地址或“自动匹配”
- 播放测试：PotPlayer 播放按钮（需本机安装）
- 保存 / 取消：提交或放弃编辑

---

## 结果页（管理与工具）

### 顶部操作
- 返回主页：回到首页（检测入口）
- 版本管理：保存 / 加载 / 删除 / 刷新（与首页一致）

### 过滤与视图
- 状态过滤：全部 / 在线 / 离线
- 类型过滤：组播 / 单播
- 设置：打开应用设置（内/外网地址、导出安全开关与 Token 等）

### 快捷工具面板
- FCC：管理时移服务器列表（添加、保存、同步、删除）
- 分组：管理分组名称与颜色（添加、保存、应用匹配、规则、删除）
- 台标：配置台标模板列表（添加、保存）
- 代理：设置“代理/外网”基址列表（添加、保存）
- 接口：生成“播放器直连接口链接”集合
  - 状态/范围下拉：选择全部/在线/离线与内网/外网
  - Token 验证开关与输入框：外网链接可启用安全令牌
  - 操作：复制 / 打开 / 播放（直连）
- EPG：管理 EPG 源（添加、保存、删除；支持 xmltv）

### 批量与分页
- 批量删除：选中后删除列表项
- 分页控制：上一页 / 下一页，支持顶部与底部控制条

### 播放与确认弹窗
- 播放测试弹窗：内置播放器预览与快速切换
- 确认弹窗：确定 / 取消 或 关闭（根据上下文显示）

---

## 版本与数据
- 保存版本：生成 streams.json 与带时间戳的版本文件（streams-YYYYMMDD-HHMMSS.json）
- 加载版本：从时间戳版本切换为当前数据
- 启动自动加载：若无 streams.json，自动加载最新的时间戳版本

---

## 导出规则（简述）
- 内网：组播使用当前 UDPXY（rtp2httpd）基址；单播保留原地址
- 外网：组播使用“外网”基址拼接 /rtp/ip:port；单播使用“代理”基址 + 去协议路径
- M3U：包含 tvg-*、group-title；支持 catchup 与 catchup-source；支持 ?fcc 参数与质量后缀

---

## 登录与安全
- 未登录：禁止访问首页与结果页，API 请求返回 401（导出与流代理除外）
- 验证码：登录需输入验证码，点击图片可刷新
- 修改密码：在设置对话框中更新用户名与密码（需要校验旧密码）

---

## 常见问题
- 数据未保存到宿主机 data：确保生产 Docker 仅映射 /app/data，宿主机目录可写；必要时在 compose 中设置 `user: "1000:1000"`
- Windows 卷挂载异常：避免中文路径，建议仓库位于英文路径（如 `C:\Work\channel-sentinel`）
- Docker 更新：版本弹窗将给出镜像更新指引（拉取最新镜像并重启）
