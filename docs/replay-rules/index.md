# 回放规则概述

Channel Sentinel 提供强大的 **回放规则引擎**，用于将直播地址转换为回放地址。

## 为什么需要回放规则？

不同地区、不同运营商的 IPTV 服务商，回放地址的格式各不相同：

| 场景 | 直播地址 | 回放地址格式 |
|------|----------|--------------|
| 湖南电信 | `http://.../index.m3u8` | `http://.../index.m3u8?starttime=xxx` |
| 广东电信 | `rtsp://.../live/...` | `rtsp://.../live/...?starttime=xxx` |
| 其他 | ... | ... |

回放规则引擎通过 **JSON 配置** 统一处理这些差异。

## 规则类型

| 类型 | 说明 | 文件 |
|------|------|------|
| 基础地址规则 | 生成回放基础地址 | `replay_base_rules.json` |
| 时间规则 | 处理时间参数格式 | `time_placeholder_rules.json` |
| 占位符规则 | 时间占位符映射 | `time_placeholder_rules.json` |

## 匹配优先级

```
1. 手动设置（最高优先级）
2. 默认基线
3. 省份规则
4. 运营商规则
5. 回退策略（使用直播地址）
```

## 工作流程

```
直播地址
    │
    ▼
┌─────────────────┐
│  基础地址规则    │ ──→ 提取基础地址，移除时间参数
└─────────────────┘
    │
    ▼
┌─────────────────┐
│   时间规则      │ ──→ 生成时间占位符
└─────────────────┘
    │
    ▼
┌─────────────────┐
│  拼接回放地址   │
└─────────────────┘
    │
    ▼
回放地址
```
