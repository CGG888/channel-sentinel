# 社区贡献

Channel Sentinel 采用 **社区驱动** 的方式维护回放规则库，任何人都可以提交规则。

## 贡献流程

```
┌──────────┐
│ 提交规则 │
└────┬─────┘
     │
     ▼
┌──────────────────┐
│ GitHub OAuth 登录 │
└────┬─────────────┘
     │
     ▼
┌──────────────────┐
│ 填写规则信息      │
│ - 省份/运营商    │
│ - M3U 行         │
└────┬─────────────┘
     │
     ▼
┌──────────────────┐
│ 发帖到 Issue #10 │
└────┬─────────────┘
     │
     ▼
┌──────────────────┐
│ 管理员审核合并    │
└────┬─────────────┘
     │
     ▼
┌──────────────────┐
│ 发布新版本规则   │
└──────────────────┘
```

## 提交规则

### 1. 关联 GitHub

在 **回放规则** 面板中点击 **关联 GitHub**。

### 2. 填写贡献表单

| 字段 | 说明 | 必填 |
|------|------|------|
| 省份 | 选择所属省份 | ✅ |
| 运营商 | 选择运营商 | ✅ |
| 城市 | 城市名称（可选）| ❌ |
| M3U 行 | 完整的 M3U URL 行 | ✅ |
| 说明 | 附加说明（可选）| ❌ |

### 3. 提交

点击提交后，系统会自动发帖到 [Issue #10](https://github.com/CGG888/channel-sentinel/issues/10)。

## Issue 评论格式

提交后会在 Issue 下生成如下评论：

```
@username 省份-运营商 提交:
时间: 2026-03-22 10:00:00
省份: 广东
运营商: 电信
城市: 广州

M3U行:
#EXTINF:-1 tvg-name="频道名" ... http://...

说明: 可选
状态: pending
```

## 规则审核

| 状态 | 说明 |
|------|------|
| `pending` | 待审核 |
| `approved` | 已通过，将合并 |
| `rejected` | 已拒绝 |

管理员审核后更新状态，合并到主规则库。

## 贡献者溯源

所有合并的规则都会保留贡献者信息，可以在 [规则详情](./rules) 页面查看。

点击贡献者名称可以跳转到原始提交评论，实现完全溯源。

## 获取最新规则

### 方式一：检查更新

在 **回放规则** 面板中点击 **检查更新**，如有新版本可以一键下载。

### 方式二：查看规则库

访问 [规则详情](./rules) 页面，查看所有可用规则。

## 注意事项

- 请确保提交的规则来源合法
- 提供准确的省份、运营商信息
- M3U 行必须是完整的直播地址
- 管理员有权拒绝不符合要求的提交
